## 1. エディタ内検索（`Ctrl+F`）

**目標**: `Ctrl+F` でインライン検索バーを開き、マッチ箇所をハイライト、`n`/`N` で前後に移動。

### AppState への追加（`honeydew/app/state.mbt`）

```moonbit
pub(all) struct AppState {
  // 既存フィールド...
  search_active : Bool        // 検索バーが開いているか
  search_input : @widget.Input // 検索文字列入力ウィジェット
}
```

`AppState::new` でも `search_active: false, search_input: @widget.Input::new()` を初期化する。

### UI への追加（`honeydew/ui/main_screen.mbt`）

`search_active` が true のとき、エディタエリアの下部 1 行を検索バーとして描画する:
- `" Search: "` ラベル + `Input` ウィジェット
- `Escape=close  n=next  N=prev` のヒントを右端に表示

エディタ自体の高さを 1 行分縮める（`@layout.Length(1)` を検索バー用に追加）。

### キーハンドリング（`honeydew/cmd/main/main.mbt`）

- `CtrlF` → Main 画面・Editor フォーカス時に `search_active = true`
- `search_active` が true のとき:
  - 文字キー・Backspace → `search_input` に渡す
  - `Enter` / `Char('n')` → 次のマッチへカーソル移動（`TextArea::find_next` を使用、なければ実装）
  - `Char('N')` → 前のマッチへカーソル移動
  - `Escape` → `search_active = false`、検索ハイライトをクリア

### TextArea への追加（`honeytui/widget/textarea.mbt`）

`find_next(query: String) -> TextArea` と `find_prev(query: String) -> TextArea` を実装する。
- 現在のカーソル位置から前方（または後方）に `query` を検索し、見つかればカーソルを移動
- 見つからなければ先頭（または末尾）からラップアラウンド
- マッチ箇所を `with_styled_lines` ハイライトで強調表示（背景色 Yellow など）

## 2. JSON エクスポート（`Ctrl+J`）

### config パッケージへの追加

**ファイル**: `honeydew/config/` 内の適切なファイル（CSV エクスポートと同じファイルが望ましい）

```moonbit
pub fn export_results_json(
  columns : Array[String],
  rows : Array[Array[String]],
) -> Result[String, String]
```

- 出力形式: `[{"col1": "val1", "col2": null}, ...]`
- `@db.null_value`（`"\u0000"`）は JSON の `null` として出力（クォートなし）
- 文字列値は `"` でエスケープ（`"` → `\"`、`\` → `\\`、改行 → `\n`）
- ファイル名は `honeydew_export_<タイムスタンプ>.json`
- 保存先は CSV と同じディレクトリ

### AppState への追加（`honeydew/app/state.mbt`）

```moonbit
pub fn AppState::export_results_json(self : AppState) -> AppState
```

`export_results_csv` と同じパターンで実装。

### キーバインド（`honeydew/cmd/main/main.mbt`）

Main 画面で `CtrlJ` を `state.export_results_json()` にバインドする。
ヘルプテキストに `Ctrl+J=JSON` を追加。

## 3. SQL ファイル読み込み（`Ctrl+O`）

**目標**: `Ctrl+O` でファイルパス入力プロンプトを表示し、`.sql` ファイルの内容をエディタに読み込む。

### AppState への追加（`honeydew/app/state.mbt`）

```moonbit
pub(all) struct AppState {
  // 既存フィールド...
  file_open_active : Bool
  file_path_input : @widget.Input
}
```

`AppState::new` で `file_open_active: false, file_path_input: @widget.Input::new()` を初期化。

`AppState::open_sql_file(path: String) -> AppState`:
- ファイルを読み込んでエディタに設定
- 失敗時はエラーステータス表示
- ファイル読み込みは既存の FFI/config パッケージの仕組みを使うか、MoonBit の標準 `@fs` を使う

### UI への追加（`honeydew/ui/main_screen.mbt`）

`file_open_active` が true のとき、画面中央にオーバーレイを描画:
- タイトル: `" Open SQL File "`
- Input ウィジェットでファイルパスを入力
- `Enter=open  Escape=cancel` のヒント

### キーハンドリング（`honeydew/cmd/main/main.mbt`）

- Main 画面で `CtrlO` → `file_open_active = true`
- `file_open_active` が true のとき:
  - 文字キー・Backspace → `file_path_input` に渡す
  - `Enter` → `state.open_sql_file(state.file_path_input.value())`
  - `Escape` → `file_open_active = false`

## 4. スキーマブラウザの拡張（indexes / triggers / functions）

**目標**: `build_schema_tree` でテーブルごとにインデックスとトリガーも表示する。

### DbConn トレイトへの追加（`honeydew/db/db.mbt`）

```moonbit
pub(open) trait DbConn {
  // 既存メソッド...
  async indexes(Self, String) -> Result[Array[String], String]   // テーブル名 → インデックス名一覧
  async triggers(Self, String) -> Result[Array[String], String]  // テーブル名 → トリガー名一覧
}
```

### 各バックエンドへの実装

**SQLite** (`honeydew/db/sqlite/`):
```sql
-- indexes
SELECT name FROM sqlite_master WHERE type='index' AND tbl_name=?

-- triggers
SELECT name FROM sqlite_master WHERE type='trigger' AND tbl_name=?
```

**PostgreSQL** (`honeydew/db/postgres/`):
```sql
-- indexes
SELECT indexname FROM pg_indexes WHERE tablename=$1 AND schemaname='public'

-- triggers
SELECT trigger_name FROM information_schema.triggers WHERE event_object_table=$1
```

**MySQL** (`honeydew/db/moon_mysql/`):
```sql
-- indexes
SELECT index_name FROM information_schema.statistics
  WHERE table_name=? AND table_schema=DATABASE() GROUP BY index_name

-- triggers
SELECT trigger_name FROM information_schema.triggers
  WHERE event_object_table=? AND trigger_schema=DATABASE()
```

### build_schema_tree の更新（`honeydew/app/state.mbt`）

各テーブルノード（depth=2）の下に:
- `"Columns"` グループ（depth=3）→ カラム一覧（depth=4）
- `"Indexes"` グループ（depth=3）→ インデックス一覧（depth=4）※空なら省略
- `"Triggers"` グループ（depth=3）→ トリガー一覧（depth=4）※空なら省略

## 6. 接続タイムアウトフィールド

**目標**: 接続フォームに timeout（秒）フィールドを追加する。

### ConnForm への追加（`honeydew/app/state.mbt`）

```moonbit
pub(all) struct ConnForm {
  // 既存フィールド...
  timeout_sec : @widget.Input  // デフォルト "10"
}
```

`field_count` / `total_fields` の計算に timeout フィールドを含める。
`route_key_to_field` でも timeout フィールドへのルーティングを追加。

### 接続時の利用（`honeydew/app/state.mbt` の `do_connect`）

現在はバックエンドの接続関数に timeout を渡していない。
フォームの値を `str_to_int` で整数に変換し、バックエンドが受け付ける場合は渡す。
バックエンドが timeout 引数をサポートしていない場合は、フォームに値を保持するだけでよい（将来の対応用）。

### UI への反映（`honeydew/ui/connect.mbt`）

接続フォームのレンダリングに `Timeout (s)` フィールドを追加。
